Topic 2 — Headings (h1 to h6)

I am h1 — The Main Title (use only once)

I am h2 — Section Heading

I am h3 — Sub-section Heading

I am h4 — Smaller Sub-heading

I am h5 — Rarely Used
I am h6 — The Smallest Heading
📝 RULES:
1. h1 only ONCE per page (SEO rule)
2. Always go in order: h1 → h2 → h3 (don't skip)
3. Don't use headings just to make text big — that's CSS's job

This text is bold and important — use <strong> for key terms or warnings.

This text is italic and emphasized — use <em> for titles, foreign words, or stress.

Search results often highlight the matched keyword using <mark>.

Main content here. Terms and conditions apply.

Water formula: H2O (subscript using <sub>)

📝 REAL WORLD USE:
<strong> → warnings, keywords, important info
<em> → book titles, technical terms
<mark> → search result highlights
<small> → copyright text, fine print
<del> → old prices, corrections

Topic 4 — Links (<a> tag)

Go to Google — basic link, opens in same tab

Open GitHub in new tab — use target="_blank" for external sites

Go to About Page — links to another file in same project folder

Jump to Contact Section below — the # links to an id on this page

Email Me — clicking this opens the email app

Call Me — clicking this dials the number on mobile

📝 RULES:
1. href is always required in <a>
2. Use target="_blank" only for external links
3. Never write "click here" — write what it goes to
4. # before an id = jumps to that section on same page

📝 RULES:
1. src = where the image is (url or file path)
2. alt = ALWAYS write this — describes image if it fails to load
3. alt also helps Google and visually impaired users (screen readers)
4. Width/height in HTML is fine for now — CSS handles it better later
5. img is a self-closing tag — no </img> needed
