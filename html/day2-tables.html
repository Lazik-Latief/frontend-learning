📝 STRUCTURE ORDER (always follow this):
<table> → <thead> → <tr> → <th> (headings)
<table> → <tbody> → <tr> → <td> (data)

th = table heading cell (bold + centered by default)
td = table data cell (normal text)

Topic 3 — Form Basics & Label Connection

📝 KEY RULES:
label "for" value must EXACTLY match input "id" value
method="GET" → data goes in URL (use for search forms)
method="POST" → data is hidden (use for login, signup, contact)
required → makes the field mandatory before submit

Topic 4 — All Input Types

📝 IMPORTANT DIFFERENCES:
checkbox → user can select MULTIPLE options (different name per group is fine)
radio → user can select ONLY ONE — all in group MUST share same name
hidden → invisible to user, sends data silently with form

Topic 5 — Select, Textarea & Buttons

📝 KEY DIFFERENCES:
<input> → self-closing, single line only
<textarea> → needs closing tag, multiline, use rows attribute for height
<select> → always wrap options in select, use value="" for placeholder option
"selected" attribute → pre-selects an option by default

Topic 6 — Important Form Attributes

📝 disabled vs readonly:
disabled → user cannot interact, value NOT sent when form submits
readonly → user can see/copy, value IS sent when form submits

value vs placeholder:
value → actual pre-filled content (gets submitted)
placeholder → just a hint, disappears when user types
