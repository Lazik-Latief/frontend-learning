Topic 1 — What is Semantic HTML?

Semantic means the tag describes the meaning of the content, not just how it looks.

Non-semantic way (wrong approach)

<div>Navigation here</div>
<div>Main content here</div>
<div>Footer here</div>

Semantic way (correct approach)

<nav>Navigation here</nav>
<main>Main content here</main>
<footer>Footer here</footer>
📝 WHY SEMANTIC HTML MATTERS:
1. SEO — Google understands your page better, ranks it higher
2. Accessibility — screen readers for blind users work correctly
3. Readability — other developers understand your code faster
4. Professional standard — every company expects this

Topic 2 — Full Page Layout Tags

<header>   → Top of page — logo, title, main nav
<nav>      → Navigation links only
<main>     → Primary page content — use ONCE
<section>  → Grouped content with a heading
<article>  → Self-contained content
<aside>    → Side or extra related content
<footer>   → Bottom of page — copyright, links

Full page structure example:

<body>
  <header>
    <h1>Site Title</h1>
    <nav>...links...</nav>
  </header>

  <main>
    <section>...about...</section>
    <article>...blog post...</article>
    <aside>...related links...</aside>
  </main>

  <footer>...copyright...</footer>
</body>
📝 RULES:
<header> and <footer> can appear inside article/section too
<main> — use only ONCE per page
<nav> — only for major navigation, not every link group

Topic 3 — section vs article vs aside

This confuses most beginners. Here is the simple rule:

section  → Part of a bigger page, needs the page for context
article  → Stands alone, makes sense even outside the page
aside    → Related but not essential to the main content

📝 QUICK REFERENCE:
<figure> + <figcaption> → image with caption
<time datetime=""> → date/time, datetime is for machines
<address> → contact info block
<abbr title=""> → abbreviation, full form shows on hover
<blockquote> → quoted content from another source
<code> → inline code snippet inside text
<pre> → preserves spaces and line breaks exactly

Topic 5 — div vs span

When NO semantic tag fits your need — use <div> or <span>. 

div  → BLOCK level  → takes full width, starts on a new line
span → INLINE level → stays inside text, no new line

div — Block Container

<!-- Two divs = two separate blocks, each on its own line -->
<div class="card">Card One</div>
<div class="card">Card Two</div>

span — Inline Container

My name is Lazik Latief and I am from Srinagar. I am learning frontend development.

Remember: the alt attribute is mandatory on every image tag. 

<!-- span stays INSIDE the paragraph, no line break -->
<p>My name is <span class="highlight">Lazik</span></p>

The key difference:

<!-- div: BLOCK — each goes to a new line -->
<div>First div</div>
<div>Second div</div>
Output:
  First div
  Second div    ← new line automatically

<!-- span: INLINE — stays on the same line -->
<p>I love <span>HTML</span> and <span>CSS</span></p>
Output:
  I love HTML and CSS   ← everything on same line
📝 WHEN TO USE WHAT:
div → grouping sections, layout boxes, cards, page wrappers
span → styling a single word, highlighting part of text

Rule: if no semantic tag fits → div for block, span for inline
Always try semantic tags FIRST before reaching for div
