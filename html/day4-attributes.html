Topic 1 — What are HTML Attributes?

Attributes give extra information about an element. They always go inside the opening tag. 

<!-- Structure -->
<tag attribute="value">content</tag>

<!-- Real examples -->
<a href="https://google.com" target="_blank">Google</a>
    ↑ attr name  ↑ attr value   ↑ second attribute

<img src="photo.jpg" alt="My photo" width="300">
      ↑ attr1          ↑ attr2        ↑ attr3

Boolean attributes — no value needed

<!-- These attributes work by just being present -->
<input type="checkbox" checked>   <!-- already ticked -->
<input type="text"     disabled>  <!-- not editable -->
<input type="text"     required>  <!-- must be filled -->
<video controls></video>          <!-- show play button -->

📝 RULES FOR ATTRIBUTES:
Always go in the OPENING tag only, never closing tag
Always use double quotes around values
Boolean attributes need no value — just write the name

Topic 2 — The id Attribute

id gives an element a unique name. Like an Aadhar card — only ONE person can have that number. Only ONE element per page can have a specific id. 

<!-- id must be UNIQUE — no two elements same id on one page -->
<header  id="main-header">...</header>
<section id="about">...</section>
<section id="projects">...</section>
<footer  id="contact">...</footer>

3 main uses of id:

<!-- USE 1: Anchor link jumps to section -->
<a href="#about">Go to About</a>
<section id="about">About content</section>

<!-- USE 2: Target with CSS using # symbol -->
<style>
  #main-header { background: #1a1a2e; color: white; }
</style>

<!-- USE 3: Target with JavaScript -->
<script>
  const header = document.getElementById("main-header");
  header.style.background = "red";
</script>
📝 id RULES:
Must be UNIQUE — only one element per id on a page
No spaces — use hyphens: id="my-section" not id="my section"
Case sensitive — id="About" is different from id="about"
Start with a letter, not a number
CSS selector: #idname (hash symbol)

Topic 3 — The class Attribute

class gives elements a shared label. Like a school uniform — many students wear it. Multiple elements CAN share the same class. 

<!-- Multiple elements sharing the SAME class -->
<div class="card">Project 1</div>
<div class="card">Project 2</div>
<div class="card">Project 3</div>

<!-- One element with MULTIPLE classes -->
<div class="card featured large">Special Project</div>
<!--              ↑ 3 classes separated by spaces -->

<!-- CSS targets ALL elements with that class -->
<style>
  .card     { background: white; border: 1px solid #ccc; }
  .featured { border-left: 4px solid gold; }
  /* . symbol = class in CSS */
</style>

<!-- JS targets ALL elements with that class -->
<script>
  const allCards = document.querySelectorAll(".card");
  // returns a list of all 3 card elements
</script>
📝 class RULES:
Many elements CAN share the same class — that is the point
One element can have MULTIPLE classes — separate with space
CSS selector: .classname (dot symbol)
JS selector: querySelectorAll(".classname")

Topic 4 — id vs class (Common Interview Question)

Feature id class
Uniqueness One element only per page Many elements can share it
CSS selector #idname (hash) .classname (dot)
JS — get one getElementById("name") querySelector(".name")
JS — get all Not applicable querySelectorAll(".name")
Best used for Unique sections, anchor links Reusable styles, components 
<!-- Using BOTH on same element — totally fine and common -->
<section id="about-section" class="section-box dark-bg">
  About content here
</section>
<!--
  id="about-section" → targets it uniquely (anchor links, JS)
  class="section-box dark-bg" → applies shared styles
-->
📝 SIMPLE RULE TO REMEMBER:
id = unique individual (like your name)
class = group label (like your job title)
You can have both on the same element — very common in real projects

Topic 5 — The data- Attribute

data- lets you store custom data inside HTML elements. Used heavily in real projects with JavaScript. You can name it anything after data-. 

<!-- Store any custom info on elements -->
<div
  class="product-card"
  data-id="101"
  data-price="999"
  data-category="electronics"
>
  iPhone Case
</div>

<!-- Access in JavaScript -->
<script>
  const card = document.querySelector(".product-card");

  console.log(card.dataset.id);        // "101"
  console.log(card.dataset.price);     // "999"
  console.log(card.dataset.category);  // "electronics"

  // dataset.anything matches data-anything in HTML
</script>

📝 data- RULES:
Name it anything: data-id, data-price, data-color, data-action
Value is always a string in HTML
Access in JS: element.dataset.id (remove "data-" prefix)
data-my-price → dataset.myPrice (hyphen becomes camelCase in JS)

REAL WORLD USE:
E-commerce → store product id and price on cards
Buttons → store what action to perform on click
Filters → store category to show or hide elements

Topic 6 — Meta Tags

Meta tags go inside <head>. Users never see them but they control how Google finds your page and how it looks when shared on WhatsApp or LinkedIn. 

<head>

  <!-- 1. charset — always FIRST, supports all characters -->
  <meta charset="UTF-8">

  <!-- 2. viewport — mobile support, always include -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- 3. description — shown in Google search results -->
  <!-- Keep under 160 characters -->
  <meta name="description" content="Lazik Latief — Frontend Developer.">

  <!-- 4. keywords — less important now but still used -->
  <meta name="keywords" content="HTML, CSS, JavaScript, frontend">

  <!-- 5. author — who made this page -->
  <meta name="author" content="Lazik Latief">

  <!-- 6. robots — tell Google how to handle this page -->
  <meta name="robots" content="index, follow">

  <!-- 7. Open Graph — how page looks on WhatsApp, LinkedIn -->
  <meta property="og:title"       content="Lazik — Frontend Dev">
  <meta property="og:description" content="My portfolio and blog.">
  <meta property="og:image"       content="https://yoursite.com/preview.jpg">
  <meta property="og:url"         content="https://yoursite.com">

  <!-- 8. title — browser tab + Google result title -->
  <title>Lazik Latief — Frontend Developer</title>

</head>
📝 WHY META TAGS MATTER FOR YOUR CAREER:
description → shown in Google search — affects how many people click
og:image → the preview image when you share portfolio on WhatsApp
og:title → the bold title shown in link previews
viewport → without this, your page looks broken on mobile
charset → without this, special characters show as weird symbols

Your portfolio site NEEDS all of these — recruiters share links.
